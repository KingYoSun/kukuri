## GitHub Actions E2E 失敗対応（React 無限再描画 / WDIO 接続 / PostCard テスト）

### 概要
GitHub Actions の desktop E2E 失敗を調査し、tauri-driver の起動待ちを追加して接続不安定を解消しました。併せて Reply/Quote の投稿経路を `usePostStore` に揃え、関連ユニットテストを更新し直しました。

### 対応内容
- tauri-driver/proxy が起動完了するまで待機するポートチェックを追加。
- ReplyForm/QuoteForm の投稿を `usePostStore.createPost` に統一（topicId 未指定時は従来の TauriApi へフォールバック）。
- PostCard/Reply/Quote 系テストの store mock と期待値を更新。
- `isE2E` 分岐を削除し、`useProfileAvatarSync` / `usePosts` / `useTopics` を通常挙動に戻す。
- `gh act` の `format-check` / `native-test-linux` を再実行し成功を確認。

### テスト/確認
