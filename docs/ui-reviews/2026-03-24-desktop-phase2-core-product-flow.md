- Summary: `apps/desktop` の core product flow を `App.tsx` 直書きから reusable component 群へ切り出した。Phase 2 の対象は topic switching、timeline header、composer、draft attachment preview、timeline/thread post rendering、author detail / follow affordance で、`DesktopApi` / Tauri invoke surface / Rust contract は維持した。`App.tsx` は引き続き orchestration owner として async load、polling、reply / thread / author selection state、media payload fetch を保持する。
- User flow summary: left nav の tracked topic から active topic を切り替え、main workspace で `View Scope` / `Compose Target` を保ったまま publish / reply できる。timeline card の click で thread context を開き、author link の click で author detail / follow action へ遷移する。image / video attachment は timeline と thread で同じ media renderer を使い、`syncing -> poster ready/image ready -> playable/unsupported` を同じ surface で表現する。
- Review result: Phase 2 implementation slice と review artifact は local で準備済み。human review と PR attach は pending。
- Shneiderman: 一貫性は topic list / composer / post card / context pane を reusable component 化して担保した。頻出操作の shortcut は topic switch, reply-in-place, thread context 再利用で維持した。informative feedback は audience label、reply banner、media status badge、relationship badge、error inline copy で維持した。internal locus of control は `post click -> thread`, `author click -> author`, `Esc` close, active topic fallback を固定することで担保した。short-term memory load は active topic, scope, audience, selected thread / author を shell 上に残す方針で評価した。
- Exceptions: private channel create/join/share/freeze/rotate、live、game、settings / diagnostics は Phase 2 の redesign 対象に含めず、legacy section のまま維持した。PR-visible preview image / short video は local Storybook review surface を基準にし、PR 作成時に添付する前提とする。
