- Figma: [desktop shell IA v2 context header and composer compacting](https://www.figma.com/design/36DZOCnFS8mq5GRMp3WJc7)。`Core/CoreProductFlow/WideWorkspace` を基準に、settings icon を `settings` に変更し、thread summary card を除去して close affordance を pane header へ移し、author pane の relationship badge を action row へ再配置し、replying close と attach spacing を compact 化した。後続 capture で `Review/DesktopShellWorkspaceGallery/WorkspacePatternsGallery` を同 file に追加し、`Channels / Live / Game / Profile overview / Profile edit` の shell patterns を review 面として追記した。

- Storybook review surface: `Shell/ShellFrame/Default`, `Shell/ShellTopBar/Default`, `Shell/ShellNavRail/Default`, `Shell/ContextPane/Default`, `Core/CoreProductFlow/WideWorkspace`, `Review/DesktopShellWorkspaceGallery/WorkspacePatternsGallery`
- Summary: shell は `left rail + main workspace + right detail pane stack` に再編した。top bar は active topic の 1 行表示だけに縮小し、status badge と settings trigger は left rail header へ移した。中央 workspace は `Timeline / Channels / Live / Game / Profile` の tab switcher と、active workspace だけを描画する 3 段構成に固定した。Profile default は overview に変え、`profileMode=edit` でのみ editor へ遷移する。profile timeline は active topic の public timeline から local author 投稿だけを抽出する。right sidebar は廃止し、thread pane と nested author pane を必要時だけ右へ積む stack に置き換えた。detail pane 背景は solid surface に統一した。backend、Tauri invoke、`DesktopApi`、domain contract は変更していない。
- User flow summary: 起動直後の reading order は `Skip to workspace -> active topic bar -> left rail topic nav -> workspace tabs -> active workspace form -> active workspace list` に整理した。desktop では context が無い限り 2 列、thread を開くと 3 列、thread 内 author を開くと 4 列になる。timeline から直接 author を開いた場合は single author pane、thread から author を開いた場合だけ `thread -> author` の stacked pane になる。mobile では nav trigger を footer に限定し、top bar は topic 表示だけに保つ。profile overview は public-facing self profile を表示し、`プロフィールを編集` から edit へ入り、保存後は overview へ戻る。
- Review result: topic-first の優先順位を shell 全体へ反映する refinement として採用。workspace switch を中央 header に集約し、left rail は topic nav と diagnostics entry に責務を限定した。right context は常設 sidebar ではなく progressive disclosure の pane stack に変わり、thread/author の導線が route と対応する形へ整理された。
- Shneiderman: 一貫性は workspace tab と route contract、left rail topic nav、detail pane stack の vocabulary を固定することで担保した。informative feedback は active topic bar、selected workspace tab、context route の正規化で維持した。error prevention は invalid `threadId` / nested `authorPubkey` の URL normalization で補った。internal locus of control は tab switching、topic switching、profile overview/edit round-trip、stacked pane close path によって維持した。short-term memory load は top bar と main lane の不要情報を削り、context を必要時だけ右へ積む形に変えることで下げた。
- Exceptions: `App.tsx` 内の orchestration は依然として大きく、workspace ごとの container 分割までは今回行っていない。manual smoke はこの implementation slice では未実施。
