# kukuri プライバシーポリシー（ドラフト）

Last updated: 2026-06-25
Legal bundle version: 1

> これはドラフトであり、法的助言ではありません。最終的な運用上・法的判断は、適切な専門家・規制当局への相談のもとで行ってください。
> 本文書は、kukuri デスクトップアプリ（client）自体に対する app-level のプライバシーポリシーです。各 community node が個別に提示するプライバシーポリシー（per-node consent）とは別建てです。

## 1. 基本方針

kukuri は P2P を基盤とし、ユーザーの identity / profile / social graph をローカルに保持します。これらは特定の中央サーバーに集約・所有されません。

## 2. ローカルに保存される情報

以下はユーザー端末のローカルにのみ保存されます。

- identity（鍵）、profile、social graph、投稿・スレッド・リアクション等のローカル projection。
- アプリ設定（表示テーマ、言語、discovery 設定、community node 設定、本ポリシーへの同意状態など）。

## 3. シグナリング・接続時に扱われる情報

P2P 接続の確立（シグナリング）や relay 経由通信の過程で、技術的に以下が扱われ得ます。

- **IP アドレス**: 直接 P2P 接続、DHT discovery、hole punching、relay 経由通信の相手方や community node / relay に対して、接続元の IP アドレスが可視となり得ます。これは P2P ネットワークで接続を成立させるために技術的に不可避なものです。
- **接続メタデータ / 接続ログ**: 接続先・接続時刻・endpoint 識別子などの接続メタデータが、経路上の community node / relay 側で一時的に扱われ得ます。

これらの取り扱い（保存期間・利用範囲など）は、接続先となる各 community node / relay の運営者のポリシーに従います。kukuri client 自体は、これらの接続メタデータを中央に収集・保存しません。

## 4. 保存期間

- ローカルに保存される情報は、ユーザーがアプリのデータを削除するまで端末に保持されます。
- community node / relay 側で扱われる接続ログの保存期間は、各運営者のプライバシーポリシーに従います。kukuri client はこれを制御しません。

## 5. 第三者提供

- kukuri client は、ユーザーの個人情報を中央集約して第三者へ提供することはありません。
- P2P の性質上、ユーザーが共有したコンテンツや接続情報は、接続相手・community node・relay に対して技術的に可視となり得ます。これは「第三者提供」ではなく、P2P 通信に内在する技術的特性です。

## 6. 診断レポート

ユーザーが任意で生成する診断レポートには、秘密鍵・認証トークン・private channel の capability secret・招待/共有トークン・DM 本文・ローカル DB パスは含まれません。

## 7. 同意

アプリ初回起動時、ネットワーク接続を開始する前に本ポリシーへの同意を確認します。同意するまで、kukuri はネットワーク接続（IP アドレスを伴う通信）を開始しません。重要な更新があった場合は再表示・再同意を求めます。
